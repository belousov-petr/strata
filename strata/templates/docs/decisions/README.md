# Decisions — {{PROJECT_NAME}}

ADRs: one decision per `ADR-NNNN-<slug>.md`, numbered sequentially, numbers never reused. Record the decision *and the options you rejected* — the rejected ones are half the value.

Format (MADR-derived):

```
# ADR-NNNN: <title>
- **Status:** accepted | superseded by ADR-NNNN
- **Date:** YYYY-MM-DD
## Context and Problem Statement
## Considered Options        (honest pros and cons)
## Decision
## Consequences              (costs too, not just wins)
```

A reversed decision gets a **new** ADR; the old one is marked superseded — never edited, never deleted. Before assigning a number, check the highest existing one.
